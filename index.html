A binary search tree is a binary tree that has additional properties: the value of the left child is less than the value of the parent, and the value of the right child is greater than the value of the parent for each tree node.
That is, the data in the binary search tree is stored in sorted form.
in C++, the associative containers set and map represent a balanced binary search tree.
A balanced binary search tree is a binary search tree with a logarithmic height.
In a balanced binary search tree, searches, insertions, and deletions take logarithmic time.
(since the path to any leaf from the root is no more than a logarithm).
In the degenerate case of an unbalanced binary search tree, such as when a sorted sequence was inserted into an empty tree, the tree would become a linear list, and the search, insertion, and deletion operations would take linear time.
Let's briefly discuss trees from the point of view of graph theory. A graph is a set of vertices and edges. An edge is a connection between two vertices.
The number of possible edges in the graph depends quadratically on the number of vertices (for understanding, you can imagine the tournament table of matches played).
A tree is a connected graph without cycles. Connectivity means that from any vertex to any other there is a path along the edges.
The absence of cycles means that this path is the only one. Traversing a graph is a systematic visit to all its vertices, once each.
There are two types of graph traversal: 1) depth-first search; 2) breadth-first search.
breadth-first search (BFS) starts from the start vertex, first visits all vertices that are one edge away from the start, then it visits all vertices two edges away from the initial one, and so on.
The breadth-first search algorithm is non-recursive (iterative) in nature.
To implement it, a queue data structure (FIFO) is used.
Depth-First Search (DFS) proceeds from the initial vertex, visiting vertices that have not yet been visited, regardless of the distance from the initial vertex.
The depth-first search algorithm is recursive in nature.
To emulate recursion in the iterative version of the algorithm, the stack data structure is used.

Perhaps the main property of hash tables is that all three insertion, lookup, and deletion operations are performed on average in O(1) time, the average search time for it is also O(1) and O(n) in the worst case.

1) Each node is colored either red or black (an additional field appears in the node data structure - a color bit).

5) Paths from a node to its leaves must contain the same number of black nodes (this is the black height).

FlyWeight - Flyweight
The Flyweight pattern shows how to effectively separate many small objects. The key concept is the distinction between internal and external states. The internal state consists of information that is context independent and can be shared (for example, the name of an icon, its width and height). It is stored in the flyweight (that is, in the Icon class). External state cannot be shared, it depends on the context and changes with it (for example, the coordinates of the top left corner for each instance of the icon). The external state is stored or computed by the client and passed to the flyweight when the operations are called. Clients should not instantiate flyweights directly, but only obtain them from the FlyweightFactory object for proper partitioning.
False sharing
When a member of the system periodically attempts to access data, which will never be changed by the other party, but this data shares a cache block with the data, that have been changed, the caching protocol can force the first participant to reload the entire block, despite the lack of logical necessity. . The caching system is unaware of the activity in this block and causes the first participant to bear the overhead of the caching system, necessary for true sharing of the resource.
