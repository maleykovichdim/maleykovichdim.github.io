probe_notification_context

Description
This interface can be used by objects implementing probes, which triggers a notifier for sampling. The interface is suitable if there are multiple notification that triggers before any probe values are read, to distinguish between the different instances.

In the notifier callback, the get method should be called by the sampler, to identify the context associated with a certain notifier.

Before the context aware probes are actually read, the corresponding set method should be called with the same context value as received earlier. 

SIM_INTERFACE(probe_notification_context) {
        lang_void *(*get)(conf_object_t *obj);
        void (*set)(conf_object_t *obj, lang_void *context);
};
#define PROBE_NOTIFICATION_CONTEXT_INTERFACE "probe_notification_context"

Execution Context
Execution Context for get method. Global context for set method.
